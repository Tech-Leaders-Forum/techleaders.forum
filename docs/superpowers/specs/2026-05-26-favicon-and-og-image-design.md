# Favicon & Social Share Image — Design Spec

## Goal

Add two brand assets to the Tech Leaders' Forum site:

1. **Favicon** — appears in browser tabs, bookmarks, and OS-level shortcuts.
2. **Open Graph image** — preview card when the URL is shared on LinkedIn, X, Slack, Discord, etc.

Both must be generated as source-controlled assets (SVG and HTML/CSS template) rather than one-off binary exports, so they can be regenerated and updated alongside the rest of the site.

## Brand constraints

Locked from the existing design system (see `README.md`):

- **Black**: `#030807` (background)
- **Lime**: `#d6ff3f` (primary accent / mark color)
- **White**: `#ffffff`
- **Display font**: Space Grotesk Bold (700)
- **Tone**: dark, high-contrast, editorial, calm. No gradients drifting into blue/purple. No beige neutrals.

No new colors, fonts, or graphic elements are introduced.

## Favicon

**Concept (chosen):** TLF monogram on a black rounded square.

- **Mark**: the letters `TLF` set in Space Grotesk Bold, lime (`#d6ff3f`) on black (`#030807`).
- **Shape**: rounded square (corner radius ~6/32 of the side, or scale appropriately at small sizes).
- **Format**: single SVG source committed to `assets/favicon.svg`.
- **Sizes shipped**:
  - `favicon.svg` — modern browsers (vector, scales everywhere)
  - `favicon-32.png` — fallback for browsers/tools that don't support SVG favicons
  - `favicon-180.png` — `apple-touch-icon` for iOS home-screen
- **Link tags in `index.html` `<head>`**:
  ```html
  <link rel="icon" type="image/svg+xml" href="assets/favicon.svg" />
  <link rel="icon" type="image/png" sizes="32x32" href="assets/favicon-32.png" />
  <link rel="apple-touch-icon" sizes="180x180" href="assets/favicon-180.png" />
  ```

**Rendering note:** at 16×16 the three letters get a bit dense. Acceptable trade — the user prioritized explicit `TLF` branding over an abstract mark (e.g., the play triangle alone).

## Open Graph (social share) image

**Layout (chosen): "Hero echo"** — mirrors the site's hero section so the share card and the page feel like the same artifact.

**Specs:**

- **Dimensions**: 1200×630 (OG / Twitter Large Card standard)
- **Format**: PNG (1200×630), generated by rendering an HTML/CSS template via headless browser
- **Source of truth**: `assets/og-image.html` (committed) — the HTML/CSS template
- **Output**: `assets/og-image.png` (committed binary)

**Content & layout** (vertically centered, padded `44px 60px` at preview scale → `88px 120px` at full 1200×630):

1. **Top label** (lime `#d6ff3f`, uppercase, letter-spacing `0.22em`, ~22px @ full size):
   > `English-language meetup · Budapest, Hungary`

2. **Wordmark** (white, Space Grotesk Bold, ~112px @ full size, line-height `0.92`, letter-spacing `-0.01em`):
   ```
   Tech
   Leaders'Forum▶
   ```
   The `▶` triangle is lime, sits inline at the end of the second line (matches the existing logo).

3. **Tagline** (Space Grotesk SemiBold ~44px @ full size, color `rgba(255,255,255,0.85)`):
   > `Conversations on leadership challenges.`

Stack vertically with `~22px` gap at preview scale (`44px` at full size). All three elements centered as a group within the canvas.

**Background:**

- Linear gradient `#030807` → `#07100e` (vertical)
- Radial glow lime/green-tinted: top-left (`#083f22` 0% → transparent 55%) and bottom-right (`#128b37` 0% → transparent 55%)
- Faint grid overlay (`rgba(255,255,255,0.025)` 1px lines every 24px, both axes)

This is the same background system used by the live site, scaled to one frame.

**Meta tags to add in `index.html` `<head>`:**

```html
<meta property="og:title" content="Tech Leaders' Forum" />
<meta property="og:description" content="An English-language, in-person forum in Budapest for engineering and tech leaders." />
<meta property="og:image" content="https://techleaders.forum/assets/og-image.png" />
<meta property="og:url" content="https://techleaders.forum/" />
<meta property="og:type" content="website" />
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:image" content="https://techleaders.forum/assets/og-image.png" />
```

Absolute URLs are required for OG — relative paths break LinkedIn/Slack previews.

## Generation approach

Both assets are produced from source files committed to the repo, so they can be regenerated without external tooling like Canva:

- **Favicon SVG**: hand-authored — small enough to write directly.
- **Favicon PNGs (32, 180)**: rendered from the SVG using a one-line script (e.g., `rsvg-convert` or a headless-browser screenshot). Documented in a `scripts/` entry so future regenerations are reproducible.
- **OG image PNG**: rendered from `assets/og-image.html` by a headless browser (Chrome/Chromium via Puppeteer, or `--screenshot` flag) at 1200×630. Also documented as a script.

The HTML template uses Google Fonts (`Space Grotesk`, already loaded by the site) — the render script must wait for fonts to load before capturing the screenshot, or the wordmark will fall back to a system font.

## Out of scope

- Multi-variant OG images per session/event (current scope: one canonical share card)
- Animated/dynamic OG (e.g., per-page generated images)
- Open Graph audit/preview tooling
- Updating the logo itself

## Success criteria

- Browser tab on `techleaders.forum` shows the lime TLF monogram, not the default globe icon
- iOS "Add to Home Screen" produces a recognizable tile (not a screenshot)
- Pasting `https://techleaders.forum` into LinkedIn, Slack, or X renders the hero-echo card with the wordmark and tagline visible at the platform's preview size
- Both source files (`favicon.svg`, `og-image.html`) and render scripts are committed; regenerating the PNGs is a single command
